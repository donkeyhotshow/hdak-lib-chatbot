# Консолидация архитектуры

## Проблема
Два отдельных фронтенда создают:
- Дублирование кода
- Разные UX паттерны
- Сложность поддержки

## Рекомендация
Выбрать один подход:

### Вариант A: Next.js (рекомендуется)
- Лучше для SEO
- Server-side rendering
- Встроенная оптимизация

### Вариант B: React SPA
- Проще деплой
- Лучше для интерактивности
- Меньше серверных ресурсов

## План миграции
1. Выбрать основную архитектуру
2. Перенести лучшие компоненты
3. Унифицировать дизайн-систему
4. Удалить дублирующий код
