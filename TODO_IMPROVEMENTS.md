# Детальный план улучшений (TODO) для HDAK Library Chatbot

На основе проведенного анализа репозитория `donkeyhotshow/hdak-lib-chatbot` и живого сайта `https://hdak-lib-chatbot.vercel.app/`, а также с учетом фокуса на мобильный UI/UX и функционал поиска по базам данных, был составлен следующий план улучшений.

## 1. Улучшения функционала поиска по базам данных

### 1.1. Мониторинг и отказоустойчивость каталога

*   **Задача:** Внедрить систему активного мониторинга доступности внешнего электронного каталога (`https://library-service.com.ua:8443/khkhdak/DocumentSearchResult`).
    *   **Описание:** Автоматически проверять доступность каталога с определенной периодичностью. В случае обнаружения проблем, уведомлять администраторов системы.
    *   **Приоритет:** Высокий.
*   **Задача:** Улучшить пользовательский опыт при недоступности каталога.
    *   **Описание:** Помимо сообщения о недоступности, предложить пользователю альтернативные действия: например, возможность оставить запрос на ручной поиск (с указанием контактных данных) или подписаться на уведомление о восстановлении работы каталога.
    *   **Приоритет:** Средний.

### 1.2. Расширение интеграции с внешними научными базами

*   **Задача:** Исследовать возможность более глубокой интеграции с внешними научными базами данных (Scopus, Web of Science, Springer Link и т.д.).
    *   **Описание:** Проверить наличие публичных API у этих сервисов. Если API доступны, разработать механизмы для выполнения поисковых запросов непосредственно из чата и получения структурированных результатов, аналогично текущей интеграции с электронным каталогом.
    *   **Приоритет:** Средний.
*   **Задача:** Уточнение намерений для рекомендаций.
    *   **Описание:** Дополнительно обучить LLM предлагать более персонализированные рекомендации на основе истории запросов пользователя, популярных тем или текущих трендов в библиотеке.
    *   **Приоритет:** Низкий.

## 2. Улучшения мобильного UI/UX

### 2.1. Тестирование и оптимизация производительности

*   **Задача:** Провести всестороннее тестирование на различных мобильных устройствах.
    *   **Описание:** Выполнить тестирование на широком спектре реальных мобильных устройств (различные производители, версии ОС, размеры экранов) для выявления и устранения возможных проблем с отображением, производительностью и взаимодействием.
    *   **Приоритет:** Высокий.
*   **Задача:** Оптимизация производительности на слабых устройствах.
    *   **Описание:** Проанализировать производительность приложения на мобильных устройствах с ограниченными ресурсами, особенно при активном использовании LLM и загрузке больших объемов данных. Оптимизировать загрузку ресурсов и рендеринг для обеспечения плавной работы.
    *   **Приоритет:** Средний.

### 2.2. Доступность (Accessibility)

*   **Задача:** Провести аудит доступности для пользователей с ограниченными возможностями.
    *   **Описание:** Убедиться, что все интерактивные элементы (кнопки, ссылки, поля ввода) имеют правильные ARIA-атрибуты, доступны с помощью клавиатуры и корректно озвучиваются скринридерами. Проверить контрастность цветов и размер шрифтов.
    *   **Приоритет:** Средний.

### 2.3. Сбор обратной связи по UI/UX

*   **Задача:** Внедрить механизм сбора обратной связи от пользователей мобильной версии.
    *   **Описание:** Добавить в интерфейс чат-бота возможность для пользователей оставлять отзывы или сообщать о проблемах, связанных с UI/UX. Это поможет выявить неочевидные проблемы и собрать предложения по улучшению.
    *   **Приоритет:** Низкий.
